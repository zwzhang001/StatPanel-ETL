# StatPanel-ETL User Guide

## 1. Purpose

StatPanel-ETL helps users build city-level statistical panel datasets from CNKI, statistical yearbook CSV files, and manually prepared data. It provides both a graphical interface and command-line tools for data collection, preprocessing, extraction, quality checking, supplementation, interpolation, transformation, and trend visualization.

For most users, the graphical interface is recommended.

## 2. Install and Start

Install dependencies from the project root:

```powershell
cd D:\CodeXTask\S-ETL\StatPanel-ETL
python -m pip install -r requirements.txt
```

Start the graphical interface:

```powershell
python src\gui\run_gui.py
```

The main window title is:

```text
StatPanel ETL
```

## 3. Prepare Input Files

Prepare a city list CSV file. A typical city list contains:

```text
ProvinceName,CityName,no
```

Example:

```csv
ProvinceName,CityName,no
Beijing,Beijing,110100
Shanghai,Shanghai,310100
```

Template files are available in:

```text
examples/templates/
```

## 4. Recommended GUI Workflow

### 4.1 Crawler

Use `Crawler` to search and collect CNKI statistical indicator data.

1. Enter `Index Keyword`, such as `GDP`.
2. Select the city list file.
3. Select an output directory.
4. Set `Start Year` and `End Year`.
5. Click `Search Indicators`.
6. Complete CNKI login, captcha, VPN, or institutional verification in the browser if required.
7. Select matched indicators in `Matched Indicators`.
8. Click `Crawl`.

For first-time use, keep `Headless Browser` unchecked so that browser login and verification can be completed manually.

### 4.2 Preprocess

Use `Preprocess` to prepare raw CSV files for extraction.

Recommended order:

1. `convert_encoding`: convert raw CSV files to UTF-8.
2. `standardize_sample`: standardize crawler sample columns.
3. `merge_by_indicator`: merge city files by indicator.

Set `Input Directory`, `Output Directory`, and `From Encoding` as needed. Chinese statistical CSV files commonly use `gb18030`.

### 4.3 Model

Use `Model` to create a city-year-indicator template.

Fill in:

- `City Name File`
- `Save Path`
- `Start Year`
- `End Year`
- `Indexes`

Use commas to separate multiple indicators:

```text
GDP,Population,Electricity
```

### 4.4 Extract

Use `Extract` to write indicator values from preprocessed samples into the model template.

Important parameters:

- `ExtractPath / Model File`: model template generated by `Model`
- `Sample File`: preprocessed sample data
- `Index Name`: target indicator name
- `Unit`: optional unit filter
- `Source Rule`: source matching rule
- `Multi-match Policy`: strategy when multiple rows match
- `Fuzzy Index`: enable fuzzy indicator-name matching

Use `Preview` and `Log` to check whether matching succeeded.

### 4.5 Quality

Use `Quality` to generate a quality report for extracted data.

Set:

- `FilePath`: extracted result file
- `Save Path`: quality report path, usually `.xlsx`
- `City Missing Threshold`
- `Index Missing Threshold`

The report helps identify missing values, weak city coverage, weak indicator coverage, and records that may need manual supplementation.

### 4.6 Supplement

Use `Supplement` to merge manually prepared values into extraction results.

The supplement file should normally include:

```text
no,Year,Index,Value,Unit,Source,Note
```

If `Allow Overwrite` is unchecked, existing non-empty values are kept and only missing values are filled.

### 4.7 Interpolation

Use `Interpolation` to fill remaining missing values.

Available rules:

- `linear`: linear interpolation
- `ffill`: forward fill
- `bfill`: backward fill
- `growth`: growth-rate-based filling

Interpolation is grouped by city and indicator so that values are not filled across unrelated records.

### 4.8 Transform

Use `Transform` to convert table formats.

- `Panel2Contab (long_to_wide)`: converts a long panel table into a wide table with years as columns.
- `Contab2Panel (wide_to_long)`: converts a wide table back into a long panel table.

### 4.9 Load

Use `Load` to generate a trend chart for quick inspection.

Set:

- `Data File`
- `Chart Save Path`
- `Variable`
- `Variable Column`
- `Year Column`
- `Value Column`
- optional `Group Column`
- optional `Chart Title`

Click `Generate Trend Chart` to save and preview the chart.

## 5. Command-Line Usage

For batch processing, run commands from `src/core`:

```powershell
cd D:\CodeXTask\S-ETL\StatPanel-ETL\src\core
python -m setl_core.cli --help
```

Common commands:

```powershell
python -m setl_core.cli model --start-year 2000 --end-year 2020 --city-file ..\..\examples\templates\city_list_template.csv --output-file model.csv --index GDP
python -m setl_core.cli convert-encoding --input-dir raw --output-dir utf8 --from-encoding gb18030
python -m setl_core.cli standardize-sample --input-dir utf8 --output-dir standard
python -m setl_core.cli merge --input-dir standard --output-dir merged
python -m setl_core.cli extract --model-file model.csv --sample-file merged\GDP.csv --output-file GDP_extract.csv --index GDP --source-mode any
python -m setl_core.cli quality --input-file GDP_extract.csv --output-path quality_report.xlsx
python -m setl_core.cli supplement --extract-file GDP_extract.csv --supplement-file supplement.csv --output-file GDP_supplemented.csv
python -m setl_core.cli interpolate --input-file GDP_supplemented.csv --output-file GDP_interpolated.csv --method linear
python -m setl_core.cli long-to-wide --input-file GDP_interpolated.csv --output-file GDP_wide.csv
python -m setl_core.cli trend --input-file GDP_interpolated.csv --output-file GDP_trend.png --variable GDP --group-column CityName
```

## 6. Troubleshooting

If the crawler cannot open the browser, check that Selenium is installed and that Firefox or a compatible browser environment is available.

If CNKI opens but no data are returned, check login status, VPN or IP authorization, captcha completion, keyword matching, city names, and waiting-time settings.

If the GUI starts with missing dependency errors, reinstall dependencies:

```powershell
python -m pip install -r requirements.txt
```

If command-line imports fail, make sure commands are run from:

```text
StatPanel-ETL/src/core
```

## 7. Output Check

After each step, check:

- `Preview`: confirms the first rows of the output.
- `Log`: records status, errors, and output paths.
- Output files: confirm CSV, Excel, or image files are created at the selected paths.

The recommended final outputs are a cleaned panel dataset, a quality report, optional supplemented/interpolated files, transformed long or wide tables, and trend charts.
